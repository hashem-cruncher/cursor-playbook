# Contributing

Pull Requests are highly welcome! If you have a highly effective prompt, a new framework rule, or a token-optimization technique, please submit a PR following the existing structure.

## How to Contribute

1. **Fork** the repository and create a branch from `main`.
2. **Match the layout** — place new content in the appropriate directory (`01-system`, `02-workflows`, `03-prompts`, `05-rules`, etc.).
3. **Follow existing style** — concise, actionable, enterprise-grade tone; no filler.
4. **Link from README** — if you add a new file, include it in the [Complete System Index](./README.md#-complete-system-index) table.
5. **Open a PR** with a clear description of what you added and why it helps Cursor users.

## What We Are Looking For

- Battle-tested **Cursor prompts** (architecture, debugging, frontend, backend, database)
- **Cursor rules files** patterns for specific frameworks or stacks
- **Token optimization** techniques that measurably reduce context usage
- Workflow improvements for feature development, code review, or legacy onboarding
- Lessons learned from real projects (hallucinations, architecture mistakes, token waste)

Thank you for helping make this playbook better for the community.
